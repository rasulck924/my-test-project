# Менің жобамдағы тест деңгейлерінің бөлінуі

Бұл жобада тест деңгейлері классикалық тест пирамидасына сәйкес бөлінген. Төменде әр деңгейдің жобадағы рөлі көрсетілген.

## 1. Модульдік тесттер (Unit tests)
Жобаның негізгі бөлігі модульдік тесттерден тұрады. Олар әр функцияның жеке-жеке дұрыс жұмыс істейтінін тексереді.  
Мысалы:
- add() функциясы сандарды дұрыс қосады ма
- is_even() функциясы санның жұп екенін дұрыс анықтай ма

Мақсаты:
- қатені ерте табу
- код өзгерген кезде тез нәтиже алу
- тесттер өте жылдам орындалады

## 2. Интеграциялық тесттер
Қазір интеграциялық тесттер жоқ, бірақ болашақта қосылады.  
Мысалы:
- Бір модуль екінші модульдің функциясын дұрыс қолдана ма
- Код бөліктерінің өзара байланысы дұрыс па

## 3. Жүйелік тесттер (System tests)
Бұл деңгей толық жүйені тексереді. Қазіргі шағын жобаға қажетті емес, бірақ үлкен жүйелерде маңызды.

## 4. Өнімділік тесттері
Өнімділік тесттері жобаның болашақ нұсқаларында қолданылады.  
Құралдар:
- JMeter, Locust, k6 – жүктеме тесті
- Grafana, Prometheus – мониторинг
- Pytest-benchmark – Python функцияларының жылдамдығын өлшеу

## Қорытынды
Бұл жобада негізгі көңіл модульдік тесттерге бөлінген. Интеграциялық, жүйелік және өнімділік тесттері жоба үлкейген сайын қосылады.
